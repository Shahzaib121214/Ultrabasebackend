computer Option 1: Local Development (XAMPP/WAMP)

info
Note: Yeh method local testing ke liye hai. Browser warning aayegi lekin aap safely bypass kar sakte hain.
1

OpenSSL Install Karein

XAMPP/WAMP mein already installed hota hai. Check karein:

C:\xampp\apache\bin\openssl.exe
2

Self-Signed Certificate Banayein

Command Prompt (Admin) mein run karein:

cd C:\xampp\apache mkdir conf\ssl.crt mkdir conf\ssl.key bin\openssl req -x509 -nodes -days 365 -newkey rsa:2048 ^ -keyout conf\ssl.key\server.key ^ -out conf\ssl.crt\server.crt

Details poochega (Country, State, etc.) - kuch bhi enter kar sakte hain

3

Apache Configuration

C:\xampp\apache\conf\httpd.conf file kholen aur uncomment karein:

LoadModule ssl_module modules/mod_ssl.so Include conf/extra/httpd-ssl.conf
4

SSL Configuration

C:\xampp\apache\conf\extra\httpd-ssl.conf file mein update karein:

Listen 443 <VirtualHost _default_:443> DocumentRoot "C:/xampp/htdocs" ServerName localhost:443 SSLEngine on SSLCertificateFile "conf/ssl.crt/server.crt" SSLCertificateKeyFile "conf/ssl.key/server.key" <Directory "C:/xampp/htdocs"> Options Indexes FollowSymLinks AllowOverride All Require all granted </Directory> </VirtualHost>
5

Apache Restart Karein

XAMPP Control Panel se Apache stop aur start karein

6

Test Karein

Browser mein open karein:

https://localhost/htdocs/
info
Warning aayegi! Click karein "Advanced" ‚Üí "Proceed to localhost (unsafe)"
Yeh safe hai kyunki aapka khud ka server hai.

public Option 2: Production (Live Website)

verified
Recommended for Production! No browser warnings, completely secure.

Prerequisites:

  • Domain name (example.com)
  • Hosting with cPanel/Plesk/Direct server access
1

Let's Encrypt SSL (Free)

cPanel Method:

  1. cPanel login karein
  2. "SSL/TLS Status" ya "Let's Encrypt SSL" option dhundein
  3. Apna domain select karein
  4. "Install" click karein
  5. Done! Automatic renew bhi hoga
2

Force HTTPS (.htaccess)

htdocs/.htaccess file banayein:

RewriteEngine On RewriteCond %{HTTPS} off RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

Yeh automatically HTTP ko HTTPS par redirect karega

android Android App mein HTTPS Use Karein

RetrofitClient Update:

object RetrofitClient { // HTTPS URL use karein private const val BASE_URL = "https://yourdomain.com/htdocs/" // Ya local testing ke liye // private const val BASE_URL = "https://192.168.x.x/htdocs/" const val API_KEY = "maxclube_secret" const val PROJECT_ID = 1 val api: UltraBaseAPI by lazy { Retrofit.Builder() .baseUrl(BASE_URL) .addConverterFactory(GsonConverterFactory.create()) .build() .create(UltraBaseAPI::class.java) } }
warning

Self-Signed Certificate ke liye:

Agar local testing mein self-signed certificate use kar rahe hain, to Android app mein network security config add karna padega. Production mein yeh zarurat nahi!
